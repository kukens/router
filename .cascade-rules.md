# Working Rules for AI Assistant

## Core Problem-Solving Principles

### 1. Simplest First Approach
- Always start with the most React-idiomatic solution
- Consider built-in React patterns before manual implementations
- Prefer declarative over imperative approaches

### 2. Present Solution Options
- Show multiple approaches when they exist
- Explain trade-offs between solutions
- Recommend the best option with clear reasoning

### 3. Question Complexity
- Stop and ask "Is there a simpler way?" before implementing complex solutions
- If solution seems over-engineered, it probably is
- Prefer less code over more code when functionality is equivalent

## Common React Patterns to Remember

### Component Reset Scenarios
- **Problem**: Need to reset component to initial state
- **Solution**: Use `key` prop to force remount
- **Example**: `<Component key={uniqueId} />`

### State Synchronization
- **Problem**: Sync state when props change
- **Solution**: Use useEffect with proper dependencies
- **Avoid**: Manual state clearing logic

### Complex State Management
- **Problem**: Multiple related state variables
- **Solution**: Consider useReducer or Context API
- **Avoid**: Excessive useState calls

### Component Lifecycle
- **Problem**: Need to run code on mount/unmount
- **Solution**: Use built-in React hooks (useEffect, useMemo, useCallback)
- **Avoid**: Class component patterns

## Communication Guidelines

### When Proposing Solutions
1. Start with the simplest approach
2. Explain why it's the best choice
3. Mention alternatives if they exist
4. Ask for confirmation before implementing

### Code Implementation
- Write minimal, focused code
- Follow existing project patterns
- Add comments only when necessary
- Prefer readability over cleverness

### Error Prevention
- Test assumptions before implementation
- Use TypeScript when available
- Handle edge cases explicitly
- Write defensive code

### File Organization
- Keep components focused on single responsibility

## Red Flags to Watch For

### Over-Engineering Indicators
- Solutions longer than 20 lines for simple problems
- Manual state management when React patterns exist
- Complex useEffect dependencies
- Multiple nested useEffects for related logic

### When to Stop and Rethink
- "I need to reset multiple state variables" → Use key prop
- "I need to sync state with props" → Check useEffect dependencies
- "I need complex state logic" → Consider useReducer
- "I need to manage component lifecycle" → Use built-in hooks
